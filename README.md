# 🖥️ ระบบสอบออนไลน์ป้องกันการทุจริตแบบ Serverless (GitHub Pages + Firebase)

ระบบสอบออนไลน์เวอร์ชันสแตนด์อโลน (Serverless) ที่โฮสต์อยู่บน **GitHub Pages** ได้โดยตรง ไม่ต้องใช้งาน Google Apps Script (GAS) หรือพึ่งพา Google Sheets อีกต่อไป โดยสลับมาใช้งาน **Firebase Firestore** ในการเก็บข้อมูลคลังข้อสอบ ผลสอบ และข้อมูลนักเรียนทั้งหมด 100%

---

## 🌟 ฟีเจอร์สำคัญของระบบ

1. **โฮสต์ฟรีบน GitHub Pages**: เข้าหน้าเว็บด้วย URL ส่วนตัว เช่น `https://ชื่อคุณ.github.io/ชื่อโปรเจกต์/`
2. **ระบบป้องกันทุจริตความปลอดภัยสูง (Proctoring)**: บล็อกคลิกขวา บล็อกก๊อปวาง บังคับเข้าเต็มจอ ตรวจจับสลับจอ/สลับแท็บ และตรวจจับสองหน้าจอเหมือนเวอร์ชันเดิม
3. **นำเข้าข้อสอบจากไฟล์ Word (.docx) สำเร็จรูป**: ถอดความข้อสอบคณิตศาสตร์ (เช่น สูตรคำนวณ ตัวเลือกแบบแนวนอนแถวเดี่ยว) ได้ 100% และบันทึกเข้าสู่ฐานข้อมูลคลาวด์ Firebase Firestore ได้ทันทีผ่านหน้าบ้านของครู
4. **จัดการตารางสอบได้บนระบบ**: ครูสามารถเพิ่มวิชาสอบ กำหนดเวลา และระบุรหัสข้อสอบได้โดยตรงผ่านแผงควบคุมบนหน้าเว็บ

---

## 🚀 ขั้นตอนการเปิดใช้งานจริงบน GitHub Pages

### ขั้นตอนที่ 1: ตั้งค่าโปรเจกต์ Firebase และ Firestore
1. ไปที่เว็บไซต์ [Firebase Console](https://console.firebase.google.com/) แล้วเข้าสู่ระบบด้วยบัญชี Google
2. คลิก **Add Project (เพิ่มโครงการ)** -> ตั้งชื่อโครงการ -> คลิกสร้างให้เสร็จสิ้น
3. ที่เมนูด้านซ้าย เลือก **Build (สร้าง)** -> **Firestore Database** -> คลิก **Create Database (สร้างฐานข้อมูล)**
   * เลือกโหมดเริ่มใช้งาน: **Start in test mode (เริ่มในโหมดทดสอบ)** ไปก่อนเพื่อให้เขียน/อ่านได้โดยไม่ต้องลงสิทธิ์
4. เมื่อสร้างเสร็จแล้ว ให้สร้าง App สำหรับหน้าเว็บ:
   * ไปที่หน้าแรกของโปรเจกต์ (Project Overview) คลิกไอคอนรูปเว็บ `</>`เพื่อสร้าง Web App
   * ตั้งชื่อแอป -> คลิก **Register App**
   * คัดลอกออบเจกต์ **`firebaseConfig`** ที่ได้ (ซึ่งจะมีค่า `apiKey`, `projectId`, `appId` เป็นต้น)

### ขั้นตอนที่ 2: ตั้งค่าคีย์เชื่อมต่อในหน้าเว็บ
1. เปิดไฟล์ [index.html](file:///d:/ระบบสอบออนไลน์%20kss/index.html) ในเครื่องคอมพิวเตอร์ของคุณ
2. ค้นหาบรรทัดที่ประมาณ 700 (ส่วนของ `const firebaseConfig = { ... }`)
3. นำค่าชุดติดตั้งที่คุณคัดลอกมาจาก Firebase ในขั้นตอนที่ 1 มาวางแทนที่
4. กดเซฟบันทึกไฟล์

### ขั้นตอนที่ 3: เปิดใช้งานเว็บแอปบน GitHub Pages
1. ส่งไฟล์ล่าสุดขึ้น GitHub (ดูรายละเอียดคำสั่ง Git ด้านล่าง)
2. เข้าไปที่คลัง Repository ของคุณบนเว็บไซต์ GitHub.com
3. คลิกแท็บ **Settings (การตั้งค่า)** ของคลังนั้น
4. เมนูด้านซ้าย เลือกหมวดหมู่ **Pages (หน้าเว็บ)**
5. ในช่อง **Build and deployment**:
   * Source: เลือก **Deploy from a branch**
   * Branch: เลือกเป็นกิ่ง **`main`** และเลือกโฟลเดอร์เป็น **`/ (root)`**
   * คลิก **Save (บันทึก)**
6. รอประมาณ 1-2 นาที ระบบของ GitHub จะแสดงลิงก์เว็บไซต์ใช้งานสำเร็จรูปที่ด้านบนของหน้านั้น เช่น `https://Sek-Skulpone.github.io/kss-online-test/`

## 🗄️ โครงสร้างคอลเลกชันใน Firebase Firestore

คุณจำเป็นต้องเข้าไปสร้างคอลเลกชัน (Collections) ตั้งต้นในแดชบอร์ด Firebase Firestore ดังนี้:

### 1. คอลเลกชันบัญชีครู: `users_teachers`
สร้างเอกสาร (Document) สำหรับบัญชีครูรายคน โดย**ตั้งชื่อ ID เอกสารเป็นชื่อสำหรับล็อกอิน (Username)**:
* **ฟิลด์ข้อมูล**:
  * `name` (ประเภท string): ชื่อครูผู้สอน
  * `password` (ประเภท string): รหัสผ่านสำหรับล็อกอิน
  * `isAdmin` (ประเภท boolean): สิทธิ์แอดมิน (`true` / `false`)

### 2. คอลเลกชันบัญชีนักเรียน: `students`
สร้างเอกสารสำหรับนักเรียนรายคน โดย**ตั้งชื่อ ID เอกสารเป็นรหัสประจำตัวนักเรียน (Username)**:
* **ฟิลด์ข้อมูล**:
  * `name` (ประเภท string): ชื่อ-สกุล นักเรียน
  * `password` (ประเภท string): รหัสผ่าน
  * `level` (ประเภท string): ชั้นเรียน เช่น `ม.4`
  * `room` (ประเภท string): ห้อง เช่น `1`
  * `no` (ประเภท string): เลขที่ เช่น `12`
  * `finishedExams` (ประเภท array): ตั้งค่าเริ่มต้นเป็นอาเรย์ว่าง `[]` (ใช้สำหรับบันทึกรหัสวิชาที่สอบเสร็จแล้ว)
  * `activeExam` (ประเภท string): ตั้งค่าเริ่มต้นเป็นค่าว่าง `""` (ใช้สำหรับล็อกห้ามเปิดหน้าต่างสอบทับซ้อน)

### 3. คอลเลกชันกำหนดการสอบ: `exams`
(คุณครูสามารถเพิ่มตารางจัดเวลาสอบได้ผ่านแผงควบคุมหน้าเว็บระบบหลังบ้านของครูได้โดยตรง) หรือสร้างฟิลด์ข้อมูลดังนี้:
* **ฟิลด์ข้อมูล**:
  * `level` (string): ระดับชั้น เช่น `ม.4`
  * `isoDate` (string): วันสอบ เช่น `2026-06-20`
  * `timeRange` (string): เวลา เช่น `08:30 - 10:00`
  * `subjectCode` (string): รหัสวิชา เช่น `MA11101`
  * `subjectName` (string): ชื่อวิชา เช่น `คณิตศาสตร์เพิ่ม1`
  * `examKey` (string): คีย์ดึงข้อสอบจากคอลเลกชัน เช่น `Q_MA11101`
  * `duration` (number): ระยะเวลาทำข้อสอบ (นาที) เช่น `60`
  * `startTimeG` (string): เวลาเริ่มสอบจริง เช่น `08:30`

*หมายเหตุ: คอลเลกชัน `questions`, `exam_results` และ `security_logs` ระบบจะทำการสร้างและบันทึกเอกสารขึ้นคลาวด์ให้อัตโนมัติเมื่อมีการอัปโหลดข้อสอบ หรือเริ่มทำข้อสอบจริง*

## 💻 คำสั่ง Git ในการอัปเดตไฟล์ขึ้น GitHub

เปิดโปรแกรม Git Bash หรือ PowerShell ในโฟลเดอร์โครงการ จากนั้นรันคำสั่งดังนี้เพื่อผลักดันโค้ดใหม่ทั้งหมดขึ้นหน้าเว็บ GitHub:

```bash
# 1. จัดคิวคิวไฟล์ลบและสร้างใหม่
git add .

# 2. คอมมิตสรุปการย้ายระบบไปคลาวด์เดี่ยว
git commit -m "Migrate server code to Firebase Firestore and deploy index.html for GitHub Pages"

# 3. ส่งข้อมูลขึ้นคลังหลัก
git push origin main
```
